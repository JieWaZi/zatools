# DevWiki Topic 编写模板

> 适用位置：`wiki/topics/<slug>.md`  
> 定位：Topic 是主题页，合并原能力边界和功能规则，不写代码实现细节。  
> 目标：让 Agent 能先判断主题是否命中，再读取核心功能机制和规则，再按需跳转 Workflow。

## 写作原则

Topic 只回答：

- 这个主题是什么；
- 功能边界是什么；
- 核心机制是什么；
- 关键规则、配置和状态是什么；
- 哪些边界、误区和联动需要注意；
- 需要看实现时跳转哪个 Workflow。

Topic 不回答：

- 代码在哪里；
- 函数怎么走；
- 接口字段怎么传；
- 怎么测试；
- UI 页面有哪些字段；
- 复杂函数内部怎么校验参数。

## explain 写作要求

允许写：

- 典型使用场景；
- 容易误解的边界；
- 低频但重要的规则；
- 与其他主题的联动说明；
- 来源、冲突、不确定和待确认问题。

禁止写：

- 验收关注点；
- 测试方法、测试位置、测试用例；
- UI 字段清单；
- 接口字段、数据库字段、代码路径；
- 参数校验细节；
- 完整产品帮助文案；
- 为了填模板而写的背景套话。

## 质量检查

- Topic 不写代码路径、函数名、handler、调用链，不写测试设计、验收清单、UI 字段清单或接口字段清单。
- 实现相关内容只写关联 Workflow 链接和一句话入口说明。
- explain 只作为 core 的低频补充。
- 不复制 raw 原文。
- 关系字段使用字符串数组。
