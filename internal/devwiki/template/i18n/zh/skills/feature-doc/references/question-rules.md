# 向用户提问的规则

以下场景允许先自主尝试几轮；仍无法确认时，必须停下来向用户提问：

## 必问场景

- 只有特性名称，没有稳定入口，且多轮检索后仍出现多个候选实现。
- 接口 URL、页面路径、关键函数在代码库里完全找不到。
- 代码通过动态注册、反射、配置下发、模板生成等方式间接生效，无法仅靠静态搜索确认。
- 关键外部系统、网关、三方接口没有本地实现或缺少文档。
- 同名文件已经存在，且需要决定“更新”还是“新建”。
- 代码与现有 wiki / raw 文档明显冲突，无法判断哪一份更新。

## 提问格式

提问要短、具体、只问缺失的锚点，不要把一串大问题丢给用户。

优先这样问：

- “`<特性名称>` 我在代码里找到两个候选入口：`a` 和 `b`。你更希望我从哪条链路开始？”
- “我没在本地代码中找到接口 `<URL>`。这个接口是否在网关、子仓库或外部服务里？”
- “当前文件只能看到调用方，看不到下游实现。你能提供关键函数名、模块名或相关接口吗？”

## 禁止的提问方式

- 禁止在还没自主搜索前就把问题丢给用户。
- 禁止泛泛地问“能多给点信息吗”。
- 禁止在已经能继续排查时频繁打断用户。
