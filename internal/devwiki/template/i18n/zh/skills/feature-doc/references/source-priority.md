# 来源优先级

默认按以下顺序使用来源：

1. `wiki/`
   - 先看已有 capability 页和 feature 页，避免重复发现已经整理过的上下文。
2. `raw/`
   - 再看需求文档、设计文档、功能说明、测试资料，判断已有资料是否覆盖当前功能。
3. 本地代码
   - 用来确认真实实现、补齐空白，并纠正文档漂移。
4. 用户澄清
   - 只有在自主调查无法继续，或必须由用户拍板时才提问。

## 使用规则

- `wiki/` 与 `raw/` 是线索和历史，不是最终真相。
- 最终结论必须落到当前代码。
- 如果代码与文档冲突，要在生成的 feature 页面中明确写出冲突。
