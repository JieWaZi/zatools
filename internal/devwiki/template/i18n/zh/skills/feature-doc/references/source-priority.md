# 信息源优先级

默认按下面顺序使用信息源：

1. `wiki/`
   - 先看已有能力文档、历史变更、历史设计，避免重复造轮子。
2. `raw/`
   - 再看需求文档、设计文档、特性说明、代码总结、复盘文档，判断已有资料是否覆盖当前特性。
3. 本地代码目录
   - 用代码确认真实实现，补齐文档缺口，修正文档漂移。
4. 用户补充信息
   - 只在自主检索无法继续、存在冲突或需要确认范围时提问。

## 使用要求

- `wiki/` 与 `raw/` 是线索和历史背景，不是最终真相。
- 最终结论要以当前代码为准；如果代码与文档冲突，要在文档中明确指出。
- 只要代码没有确认，不要把历史文档里的描述直接写成事实。
