# Trace Playbook

- Check wiki pages first, then raw sources, then code, then ask the user.
- Prefer a concrete entry anchor: API URL, route, key file, key function, or known feature name.
- Search `wiki/` and `raw/` for matching capabilities, feature pages, requirements, and designs before expanding into code.
- When code is needed, start from entry points and follow enough of the flow to explain entry points, call chain, key logic, state reads/writes, and change impact.
- Stop tracing once the workflow page can explain engineering location and implementation clues without guessing.
